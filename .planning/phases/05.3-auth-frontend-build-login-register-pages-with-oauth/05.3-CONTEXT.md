# Phase 05.3: Auth Frontend - Context

**Gathered:** 2026-03-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Build frontend authentication UI — login page, register page, AuthContext for state management, and Google OAuth integration. Backend auth endpoints already exist (FastAPI Users). This phase delivers:
- Login and register pages with email/password forms
- Google OAuth "Sign in with Google" flow
- AuthContext to manage auth state across the app
- Navbar integration (login button when logged out, avatar dropdown when logged in)
- Protected route handling for /admin

</domain>

<decisions>
## Implementation Decisions

### Auth Flow UX
- Dedicated pages at /login and /register (not modals or panels)
- Form validation errors appear inline under each field (red text)
- After login, redirect back to previous page (remember intent)
- "Remember me" checkbox: checked = 30 days, unchecked = session only

### OAuth Providers
- Google only (no Microsoft or GitHub for v1)
- Google button above email form with "or" divider between them
- Auto-create account on first Google login (zero-friction onboarding)
- Google branded button (white with 'G' logo, follows Google guidelines)

### Protected Routes
- /analyze: Allow without login, show "Create account to save history" prompt after results
- /admin: Hide nav link for non-admins, 404 if they navigate directly
- /glossary: Public, no login required
- On 401 API response: Show "Session expired" toast, then redirect to /login with return URL

### Auth State UI (Navbar)
- Logged out: Show "Login" text button (not competing with main CTA)
- Logged in: Show avatar + dropdown menu
- Avatar fallback: User initials on colored background (no Gravatar)
- Dropdown items: Profile, My Analyses (history), Logout

### Claude's Discretion
- Token storage mechanism (localStorage vs httpOnly cookie)
- Exact form field validation timing (on blur vs on submit)
- Avatar background color generation (hash-based vs random)
- Loading states during auth operations
- Exact toast styling and duration

</decisions>

### Established Patterns
- i18n: `useTranslations()` hook with messages in `frontend/messages/{locale}.json`
- RTL: `dir={isHebrew ? 'rtl' : 'ltr'}` on containers
- Animation: Framer Motion `AnimatePresence` for page transitions
- State: useState only, no global state management (keep simple)
- API pattern: async functions in `client.ts`, transform responses
