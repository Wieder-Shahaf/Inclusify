<task type="auto">
  <name>Task 3: Create docker-compose.yml with profiles</name>
  <files>docker-compose.yml, .env.example</files>
  <action>
Create `docker-compose.yml` at project root with services:

**postgres service:**
- Image: `postgres:16-alpine`
- Profile: `["dev"]` only (prod uses Azure PostgreSQL)
- Environment: POSTGRES_DB=inclusify, POSTGRES_USER=postgres, POSTGRES_PASSWORD=devpassword
- Volumes: `postgres_data:/var/lib/postgresql/data`, mount `./db/schema.sql` and `./db/seed.sql` to `/docker-entrypoint-initdb.d/`
- Port: 5432:5432
- Healthcheck: `pg_isready -U postgres` with 5s interval, 5 retries

**backend service:**
- Profiles: `["dev", "prod"]`
- Build context: `.`, dockerfile: `infra/docker/backend.Dockerfile`, target: `${BUILD_TARGET:-development}`
- Environment: PGHOST, PGPORT, PGDATABASE, PGUSER, PGPASSWORD, PGSSL (from env vars with defaults for dev)
- Ports: 8000:8000
- Dev volumes: `./backend:/app` for hot-reload
- depends_on postgres (condition: service_healthy, only in dev)
- Healthcheck: curl localhost:8000/health (interval 10s)

**frontend service:**
- Profiles: `["dev", "prod"]`
- Build context: `.`, dockerfile: `infra/docker/frontend.Dockerfile`, target: `${BUILD_TARGET:-development}`
- Environment: NEXT_PUBLIC_API_URL (default http://localhost:8000)
- Ports: 3000:3000
- Dev volumes: `./frontend:/app`, `./frontend/node_modules` (anonymous volume to preserve container node_modules)
- depends_on backend

Define named volume: `postgres_data`

Create `.env.example`:
```
# Database (local dev defaults work with docker compose)
PGHOST=postgres
PGPORT=5432
PGDATABASE=inclusify
PGUSER=postgres
PGPASSWORD=devpassword
PGSSL=

# Frontend
NEXT_PUBLIC_API_URL=http://localhost:8000

# Build
BUILD_TARGET=development
```

Per CONTEXT.md: single compose file with profiles, hot-reload via volume mounts.
