## Architecture Decisions

### Batch API Integration
- **Pattern:** Claude Message Batches API with polling
- **Rationale:** 50% cost savings vs synchronous API, acceptable 24h latency for dataset generation
- **Trade-off:** Cannot get real-time results, requires polling infrastructure

### Stratified Generation
- **Pattern:** Maintain existing class proportions when scaling to 11K
- **Rationale:** Prevents introducing class imbalance that could bias model training
- **Implementation:** `proportion = current_count / total → target = proportion * 11000`

### Prompt Variation Strategy
- **Pattern:** Template-based variation with randomized domain/framing
- **Rationale:** Simple synonym substitution creates near-duplicates. Domain and framing changes create true semantic diversity.
- **Example:** "Homosexual lifestyle is a choice" → "Research suggests sexual orientation can be influenced by environment and upbringing" (same incorrect claim, different academic framing)

### Temperature Selection
- **Value:** 0.9 (higher than default 0.7)
- **Rationale:** Dataset synthesis prioritizes diversity over consistency. Higher temperature prevents repetitive patterns.
- **Risk mitigation:** Schema validation filters out incoherent outputs
