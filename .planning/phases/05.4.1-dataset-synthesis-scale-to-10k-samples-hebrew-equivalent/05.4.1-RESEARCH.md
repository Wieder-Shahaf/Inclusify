## Summary

This phase requires scaling the existing 1,001-sample English dataset to 10,000 high-quality samples and creating a culturally-appropriate Hebrew equivalent. Research shows LLM-based synthetic data generation (using GPT-4 or Claude APIs) is the current industry standard for this task, with well-established quality metrics and deduplication techniques.

**Key findings:**
- Batch APIs from OpenAI and Anthropic offer 50% cost reduction for bulk generation (critical for 9,000 samples)
- Semantic deduplication using sentence-transformers with cosine similarity threshold (0.85) is standard for quality control
- Hebrew presents unique challenges: morphologically rich language, gender-based grammar, and evolving LGBTQ+ terminology
- Manual review sampling (5-10% of synthetic data) is industry best practice for quality validation
- Stratified sampling must maintain class distribution (~20% appropriate, ~80% problematic) during generation

**Primary recommendation:** Use OpenAI Batch API or Claude Message Batches API for cost-efficient generation, sentence-transformers for multilingual semantic deduplication, and stratified manual review for quality validation.

### Pattern 1: Batch API Processing with Cost Optimization
**What:** Use asynchronous batch APIs to generate thousands of samples with 50% cost savings
**When to use:** When generating >1000 samples and 24-hour latency is acceptable
**Example:**
```python
# Source: OpenAI Batch API docs (https://platform.openai.com/docs/api-reference/batch)
import openai
from openai import OpenAI

client = OpenAI()

# Create batch input file (JSONL format)
batch_requests = [
    {
        "custom_id": f"request-{i}",
        "method": "POST",
        "url": "/v1/chat/completions",
        "body": {
            "model": "gpt-4o",
            "messages": [
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": f"Generate variation of: {sentence}"}
            ],
            "max_tokens": 500
        }
    }
    for i, sentence in enumerate(existing_samples)
]

# Upload batch file
with open("batch_input.jsonl", "w") as f:
    for req in batch_requests:
        f.write(json.dumps(req) + "\n")

batch_file = client.files.create(
    file=open("batch_input.jsonl", "rb"),
    purpose="batch"
)

# Create batch job (results in ~24 hours, 50% discount)
batch = client.batches.create(
    input_file_id=batch_file.id,
    endpoint="/v1/chat/completions",
    completion_window="24h"
)

# Poll for completion
while batch.status not in ["completed", "failed"]:
    time.sleep(60)
    batch = client.batches.retrieve(batch.id)

# Download results
result_file = client.files.content(batch.output_file_id)
results = [json.loads(line) for line in result_file.text.split('\n') if line]
```

### Pattern 2: Semantic Deduplication with Sentence Embeddings
**What:** Compute multilingual embeddings and filter near-duplicates using cosine similarity
**When to use:** After batch generation, before validation, for both English and Hebrew datasets
**Example:**
```python
# Source: Sentence-Transformers docs + NVIDIA NeMo Curator patterns
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

# Load multilingual model (supports Hebrew + English)
model = SentenceTransformer('sentence-transformers/all-MiniLM-L6-v2')

def deduplicate_by_semantic_similarity(sentences, threshold=0.85):
    """Remove near-duplicate sentences using cosine similarity."""
    # Generate embeddings
    embeddings = model.encode(sentences, show_progress_bar=True)

    # Compute pairwise cosine similarity
    similarity_matrix = cosine_similarity(embeddings)

    # Keep track of sentences to remove
    to_remove = set()

    for i in range(len(sentences)):
        if i in to_remove:
            continue
        for j in range(i+1, len(sentences)):
            if j in to_remove:
                continue
            if similarity_matrix[i][j] > threshold:
                to_remove.add(j)  # Remove later occurrence

    # Return deduplicated sentences
    unique_indices = [i for i in range(len(sentences)) if i not in to_remove]
    return [sentences[i] for i in unique_indices], len(to_remove)

# Usage
unique_samples, removed_count = deduplicate_by_semantic_similarity(
    generated_sentences,
    threshold=0.85
)
print(f"Removed {removed_count} near-duplicates, kept {len(unique_samples)}")
```

### Pattern 3: Stratified Generation for Class Balance
**What:** Generate synthetic samples proportionally to target class distribution
**When to use:** When expanding dataset while maintaining ~20/80 class split
**Example:**
```python
# Source: Scikit-learn stratified sampling + synthetic data generation best practices
import pandas as pd
from sklearn.model_selection import train_test_split

def stratified_generation_plan(existing_df, target_total=10000):
    """Calculate how many samples to generate per class."""
    current_counts = existing_df['Severity Label'].value_counts()
    current_total = len(existing_df)

    # Target distribution (maintain proportions)
    target_distribution = current_counts / current_total
    target_counts = (target_distribution * target_total).astype(int)

    # Samples to generate per class
    generation_plan = target_counts - current_counts

    return generation_plan.to_dict()

# Example output
# {'Correct': 1800, 'Outdated': 2400, 'Biased': 3200, ...}
# Total = 10000, preserves ~20% appropriate, ~80% problematic

# Then generate in batches per class
for severity_label, count_needed in generation_plan.items():
    # Filter existing samples of this class
    class_samples = existing_df[existing_df['Severity Label'] == severity_label]

    # Generate variations
    for sample in class_samples.sample(n=min(count_needed, len(class_samples))):
```

```python
from sklearn.model_selection import StratifiedShuffleSplit

def sample_for_manual_review(df, n_samples=100, random_state=42):
```

### Anti-Patterns to Avoid

- **Sequential API calls without batching:** Costs 2x vs batch API and risks rate limits with 9K requests
- **Exact string matching for deduplication:** Misses paraphrases ("same-sex couples" vs "gay parents"); use semantic similarity
- **Uniform random sampling from existing dataset:** Breaks class distribution; always use stratified sampling
- **Translating with Google Translate only:** Produces literal Hebrew, misses cultural adaptation (e.g., gendered language, LGBTQ+ terms)
- **No manual review checkpoint:** LLMs hallucinate; 95%+ quality requires human validation sampling

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| LLM API batching | Custom async queue | OpenAI Batch API / Claude Message Batches | 50% cost savings, built-in rate limiting, persistent job state |
| Multilingual embeddings | Train custom model | sentence-transformers (all-MiniLM-L6-v2, LaBSE) | Pre-trained on 100+ languages including Hebrew, proven effective |
| Pairwise similarity at scale | Nested loops | sklearn.metrics.pairwise.cosine_similarity | Vectorized numpy operations, 100x faster |
| Hebrew translation | Character-by-character logic | Google Translate API + human review | Morphologically rich language with complex gender rules, requires linguistic expertise |
| Stratified sampling | Manual class filtering | sklearn.model_selection.StratifiedShuffleSplit | Ensures proportional class representation, handles edge cases |

**Key insight:** Synthetic data generation has matured significantly in 2025-2026. The hard problems (semantic similarity, multilingual embeddings, batch processing) are solved by established libraries. Focus on domain-specific prompts and cultural adaptation, not infrastructure.

## Common Pitfalls

### Pitfall 1: Insufficient Prompt Engineering for Diversity
**What goes wrong:** LLM generates repetitive variations ("LGBTQ+ people" → "LGBTQ+ individuals" → "LGBTQ+ persons") instead of semantic diversity (different contexts, framing, academic domains)
**Why it happens:** Generic prompts like "paraphrase this sentence" produce surface-level lexical changes
**How to avoid:**
- Specify semantic variation types: "change academic domain (psychology → sociology)", "shift framing (neutral → activist)", "vary terminology (clinical → community)"
- Include few-shot examples showing diverse contexts
- Randomize prompt variations across batch requests
**Warning signs:** High cosine similarity (>0.90) after deduplication; manual review shows repetitive phrasing

### Pitfall 2: Class Imbalance Drift During Generation
**What goes wrong:** Final dataset has 30% appropriate vs target 20%, breaking training assumptions
**Why it happens:** LLMs prefer generating "correct" examples over "problematic" ones (alignment training bias)
**How to avoid:**
- Generate each class separately with class-specific prompts
- Monitor class counts during generation
- Over-generate problematic classes by 10-20% buffer, then downsample to exact proportions
**Warning signs:** Stratified split validation shows >2% deviation from target distribution

### Pitfall 3: Hebrew Translation Without Cultural Adaptation
**What goes wrong:** Literal Hebrew translations lack natural phrasing, misuse LGBTQ+ terminology, or default to masculine forms
**Why it happens:** Machine translation optimizes for grammatical correctness, not cultural context; Hebrew LGBTQ+ terms are evolving and not in standard training data
**How to avoid:**
- Consult with Achva LGBT organization for Hebrew terminology glossary
- Use gender-inclusive Hebrew patterns (e.g., -imot/-otim suffixes for mixed-gender groups)
- Human review by native Hebrew speaker with LGBTQ+ domain knowledge
- Cross-reference with Israeli LGBTQ+ publications and academic style guides
**Warning signs:** Translated text uses only masculine forms, archaic terms like "homosexualism", or English loanwords where Hebrew terms exist

### Pitfall 4: Over-Aggressive Deduplication
**What goes wrong:** Setting similarity threshold too low (e.g., 0.70) removes legitimate variations, shrinking dataset below 10K target
**Why it happens:** Fear of near-duplicates leads to conservative threshold; pairwise comparison exponentially grows with dataset size
**How to avoid:**
- Start with 0.85 threshold (industry standard), only lower if manual review shows true duplicates
- Deduplicate in stages: exact duplicates (1.0) → near-duplicates (0.85+)
- Generate 10-15% buffer (11K samples) to account for deduplication losses
**Warning signs:** Deduplication removes >15% of samples; manual inspection shows removed samples are semantically distinct

### Pitfall 5: Batch API Job Failures Without Retry Logic
**What goes wrong:** Batch job fails after 20 hours, losing all progress; no way to resume
**Why it happens:** API timeouts, token limits exceeded, malformed JSONL input
**How to avoid:**
- Validate JSONL format before upload (no trailing commas, proper escaping)
- Split large batches into chunks (e.g., 2000 requests/batch, 5 batches total)
- Implement polling with timeout + failure handling
- Save intermediate results after each batch completes
**Warning signs:** Batch status returns "failed" or "cancelled"; error logs show "invalid request format"

### Example 2: Multilingual Diversity Metric
```python
# Source: Sentence-transformers + diversity evaluation best practices
# https://amitness.com/posts/diversity-evals/
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

def calculate_diversity_score(sentences, model_name='all-MiniLM-L6-v2'):
    """
    Compute average pairwise dissimilarity as diversity metric.
    Target: >0.7 for high diversity (0.3 average similarity).
    """
    model = SentenceTransformer(model_name)
    embeddings = model.encode(sentences)

    # Pairwise cosine similarity
    similarity_matrix = cosine_similarity(embeddings)

    # Extract upper triangle (exclude diagonal and duplicates)
    n = len(sentences)
    upper_triangle_indices = np.triu_indices(n, k=1)
    pairwise_similarities = similarity_matrix[upper_triangle_indices]

    # Diversity = 1 - average similarity
    avg_similarity = pairwise_similarities.mean()
    diversity_score = 1 - avg_similarity

    return diversity_score, avg_similarity

# Usage
diversity, avg_sim = calculate_diversity_score(generated_dataset)
print(f"Diversity score: {diversity:.3f} (target: >0.7)")
print(f"Avg pairwise similarity: {avg_sim:.3f} (target: <0.3)")
```

### Example 3: Hebrew Language Validation
```python
# Source: langdetect library + Hebrew NLP best practices
from langdetect import detect, detect_langs
import pandas as pd

def validate_hebrew_dataset(df, text_column='Sentence'):
    """
    Ensure Hebrew dataset contains Hebrew text, not English contamination.
    Flags: Mixed-language samples, RTL formatting issues, encoding errors.
    """
    issues = []

    for idx, row in df.iterrows():
        text = row[text_column]

        # Language detection
        try:
            detected_lang = detect(text)
            if detected_lang != 'he':
                issues.append({
                    'row': idx,
                    'issue': 'wrong_language',
                    'detected': detected_lang,
                    'text': text[:100]
                })
        except:
            issues.append({
                'row': idx,
                'issue': 'detection_failed',
                'text': text[:100]
            })

        # Check for Hebrew characters (UTF-8 range 0x0590-0x05FF)
        hebrew_chars = sum(1 for c in text if '\u0590' <= c <= '\u05FF')
        total_chars = len([c for c in text if c.isalpha()])

        if total_chars > 0 and hebrew_chars / total_chars < 0.8:
            issues.append({
                'row': idx,
                'issue': 'low_hebrew_ratio',
                'ratio': hebrew_chars / total_chars,
                'text': text[:100]
            })

    return pd.DataFrame(issues)

# Usage
validation_report = validate_hebrew_dataset(hebrew_df)
if len(validation_report) > 0:
    print(f"Found {len(validation_report)} issues:")
    print(validation_report)
else:
    print("✓ All samples validated as Hebrew")
```
