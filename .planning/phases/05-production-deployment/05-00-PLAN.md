<task type="auto">
  <name>Task 1: Create Azure infrastructure validation script</name>
  <files>scripts/validate-azure.sh</files>
  <action>
Create a bash script to validate Azure infrastructure for Phase 5 deployment.

Script must check:

1. Azure CLI authentication:
   - `az account show` succeeds
   - Display current subscription

2. Resource Group exists:
   - `az group show --name inclusify-rg`

3. Container Registry exists and accessible:
   - `az acr show --name inclusifyacr --resource-group inclusify-rg`

4. PostgreSQL server exists and accessible:
   - `az postgres flexible-server show --name inclusify-db --resource-group inclusify-rg`

5. Container Apps Environment exists (after 05-01 runs):
   - `az containerapp env show --name inclusify-env --resource-group inclusify-rg`
   - Exit gracefully if not yet created (script may run before deployment)

6. Backend Container App exists and healthy (after 05-01 runs):
   - `az containerapp show --name inclusify-backend --resource-group inclusify-rg`
   - Check provisioningState is "Succeeded"
   - Get FQDN and test health endpoint

7. Frontend Container App exists (after 05-01 runs):
   - `az containerapp show --name inclusify-frontend --resource-group inclusify-rg`
   - Check provisioningState is "Succeeded"

8. Secrets verification (INFRA-02 requirement):
   - `az containerapp secret list --name inclusify-backend --resource-group inclusify-rg`
   - Verify db-password and jwt-secret exist as Container Apps secrets
   - Verify secrets are NOT exposed in environment variables (check env vars don't contain raw secret values)

Exit codes:
- 0: All checks passed
- 1: Pre-deployment checks failed (ACR, PG, RG)
- 2: Post-deployment checks failed (Container Apps)
- 3: Secrets verification failed

Script should support `--pre-deploy` flag to only run pre-deployment checks (RG, ACR, PG).

Make script executable: chmod +x scripts/validate-azure.sh
  </action>

  <files>scripts/test-e2e-flow.sh</files>
  <action>
Create a bash script to test the complete E2E flow: upload PDF -> analysis -> results.

Script must:

1. Accept parameters:
   - BACKEND_URL (required or auto-detect from Azure)
   - FRONTEND_URL (required or auto-detect from Azure)
   - TEST_PDF_PATH (optional, default to test fixture)

2. Auto-detect URLs from Azure if not provided:
   ```bash
   BACKEND_FQDN=$(az containerapp show --name inclusify-backend --resource-group inclusify-rg --query "properties.configuration.ingress.fqdn" -o tsv)
   BACKEND_URL="https://${BACKEND_FQDN}"
   ```

3. Health check both services:
   - `curl -sf "${BACKEND_URL}/health"` - must return 200
   - `curl -sf "${FRONTEND_URL}"` - must return 200 (HTML)

4. Test backend analysis endpoint directly:
   ```bash
   # Use inline test text (no PDF required for basic test)
   curl -X POST "${BACKEND_URL}/api/v1/analysis/analyze" \
     -H "Content-Type: application/json" \
     -d '{"text": "The homosexual lifestyle is deviant.", "language": "en", "private_mode": false}'
   ```
   - Must return 200 with JSON containing "issues" array
   - Must detect at least one issue in the test text

5. Test PDF upload endpoint (if test PDF exists):
   ```bash
   curl -X POST "${BACKEND_URL}/api/v1/ingestion/upload" \
     -F "file=@${TEST_PDF_PATH}"
   ```
   - Must return 200 with extracted text

6. Output summary:
   - Health checks: PASS/FAIL
   - Analysis endpoint: PASS/FAIL
   - Upload endpoint: PASS/FAIL (or SKIP if no test PDF)
   - Total: X/Y tests passed

Exit codes:
- 0: All tests passed
- 1: Health checks failed
- 2: Analysis endpoint failed
- 3: Upload endpoint failed

Make script executable: chmod +x scripts/test-e2e-flow.sh
