## Spacing Scale

Declared values (must be multiples of 4). All values match the Tailwind 4-point grid already used throughout the project.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`, form field `mb-4`) |
| lg | 24px | Section padding (`p-6`), card inner padding |
| xl | 32px | Layout gaps between major sections (`gap-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- Modal inner padding: 32px (`p-8`) — matches `ProfileSetupModal` pattern
- Admin card inner padding: 24px (`p-6`) — standard card padding for `FrequencyTrendsCard` and admin dashboard cards

Source: existing codebase patterns in `ProfileSetupModal.tsx`, `globals.css`

## Typography

All sizes reference CSS px equivalents. The project uses Tailwind utility classes; the table maps utility names to values for executor reference.

| Role | Size | Tailwind class | Weight | Weight class | Line Height |
|------|------|----------------|--------|--------------|-------------|
| Body | 14px | `text-sm` | 400 (regular) | `font-normal` | 1.5 |
| Label | 14px | `text-sm` | 400 (regular) | `font-normal` | 1.25 |

| Display | 24px | `text-2xl` | 700 (bold) | `font-bold` | 1.2 |

Exact weights in use: **regular (400)** and **bold (700)**. No other weights are part of this design contract. Label differentiation from body is achieved via color contrast (muted vs. default text color), not weight.

## Color

Project uses a custom token system defined in `globals.css` on top of shadcn CSS variables. The 60/30/10 split maps as follows:

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#ffffff` / dark `oklch(0.145 0 0)` | `--background` | Page background, modal overlay body |
| Secondary (30%) | `oklch(1 0 0)` / dark `oklch(0.205 0 0)` | `--card` | Cards (`rounded-xl border bg-white dark:bg-slate-900`), modal glass surface, admin dashboard cards, form input backgrounds |
| Accent (10%) | `#7b61ff` | `--color-pride-purple` | CTA buttons, active nav links, focus rings, chart primary color, form field focus ring |
| Accent-secondary | `#ff53a1` | `--color-pride-pink` | CTA gradient end color (always paired with purple in `btn-primary`) |
| Destructive | `oklch(0.577 0.245 27.325)` | `--destructive` | Form validation errors (`text-red-500`), required field asterisks |
| Warning / Amber | `amber-50 / amber-700` | Tailwind utility | `HealthWarningBanner variant='error'` — LLM-down results banner |
| Info / Sky | `sky-50 / sky-700` | Tailwind utility | `HealthWarningBanner variant='info'` — pre-analysis model-offline banner |

Accent (`--color-pride-purple`, `#7b61ff`) is reserved for:
1. Primary CTA button gradient fill (paired with `#ff53a1`)
2. Active navigation link text (`text-pride-purple`)
3. Form input focus ring (`focus:ring-pride-purple`)
4. Bar chart bars in `SimpleBarChart` (D-05) — use `color='#7b61ff'` matching `SimpleLineChart` default
5. Section header icons in admin dashboard cards

Accent is NOT to be used for: body text, card borders, background fills, badge backgrounds, or secondary labels.

Glass surface pattern (modals): `bg-white/60 dark:bg-slate-900/50 backdrop-blur-md border border-white/40 dark:border-slate-800/50` — the `.glass` utility class.

Source: `globals.css` custom tokens, `ProfileSetupModal.tsx`, `OverviewTab.tsx`, `HealthWarningBanner.tsx`

## Interaction Contracts

### D-01: Profile Completion Popup
- **Trigger:** `useEffect` on `user` object change. Opens when `!dismissed && (!user.full_name || !user.institution || !user.profession)`.
- **Dismissal:** Clicking X, `t('profile.setup.skip')` ("Skip for now"), or submitting successfully all call `dismiss()` (sets `sessionStorage['profile_setup_dismissed']`).
- **Form fields:** All three (`full_name`, `institution`, `profession`) show required asterisk `<span className="text-red-500">*</span>`.
- **Submit state:** Button label changes from `t('profile.setup.save')` → `t('profile.setup.saving')` during async submit. Button disabled while submitting.
- **Success:** `toast.success(t('success'))` fires. `dismiss()` called (not `setOpen(false)`) to prevent re-open on `refreshProfile()`.
- **Error:** `toast.error(t('error'))` fires. Modal stays open.
- **Focus management:** `autoFocus` on `full_name` field. Radix Dialog provides focus trap.

### D-02: LLM-Down Banner (Results View)
- **Trigger:** Render `<HealthWarningBanner>` when `viewState === 'results' && analysisMode === 'rules_only'`.
- **Position:** Immediately above the results content, below the results page header. Full-width, no border radius (banner style, `border-b`).
- **Animation:** `framer-motion` entry `opacity: 0 → 1, y: -10 → 0` (already in `HealthWarningBanner`).
- **Props:** `variant='error'` (amber), `linkHref={/${locale}/glossary}`, `linkText={t('analyzer.llmDownResultsLink')}`.
- **This banner is distinct** from the pre-analysis model-offline banner (lines 357–364 in `analyze/page.tsx`) — different i18n keys, different condition.

### D-03: PDF Report Watermark
- **Placement:** Bottom-center footer of **every page**, inserted inside the existing per-page loop after `autoTable` finishes.
- **Style:** fontSize 8pt, font 'helvetica' normal, textColor RGB(150, 150, 150) — `doc.setTextColor(150, 150, 150)`.
- **Position:** `doc.text(text, pageWidth / 2, pageHeight - 8, { align: 'center' })`.
- **Locale-aware text:**
  - EN: `"Achva LGBTQ+ Studential Organization"`
  - HE: `"ארגון אחווה להט״ב הסטודנטיאלי"`
- **Base64 return mode:** `exportReport(analysis, { fileName, locale, returnBase64: true })` returns `doc.output('datauristring')` instead of triggering download. Callers without `returnBase64` (default `false`) are unaffected.

### D-04: Contact Us Modal
- **Navbar button:** Added in the right-side action cluster of `Navbar.tsx`, between `ThemeToggle` and the user/login zone. Uses `.btn-ghost` class. Label: `{t('contact.button')}` = `"Contact Us"`. Always visible on all pages.
- **Analyze page button:** Existing results-header button stays but its `onClick` is replaced: remove `handleContactUs` mailto shortcut, use `() => setContactOpen(true)` instead.
- **Modal structure** (follows `ProfileSetupModal` exactly):
  - Overlay: `fixed inset-0 z-50 bg-black/40 backdrop-blur-sm`
  - Content: `fixed z-50 left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2 w-full max-w-md`
  - Inner: `.glass p-8 rounded-2xl shadow-2xl`
  - Title: `text-xl font-bold text-slate-900 dark:text-slate-100`
  - Subtitle: `text-sm text-slate-500 dark:text-slate-400 mt-1`
- **Read-only prefilled fields:** Name, email, institution — rendered as `<p>` or disabled `<input>` with `text-slate-500` muted style. Not part of Zod schema validation.
- **Editable fields:** Subject (text input), Message (textarea, min-height 96px / `min-h-24`).
- **Attachment indicator:** If analysis is complete, show read-only info row: checkmark icon + `{t('contact.attachReport')}` = `"Analysis report attached"`. Text `text-sm text-green-600 dark:text-green-400`. Icon: `lucide-react CheckCircle` (w-4 h-4).
- **Send button:** `.btn-primary flex-1`. Label `{t('contact.send')}` / `{t('contact.sending')}` during submit. Disabled while submitting.
- **Success toast:** `toast.success(t('contact.success'))` = `"Message sent!"`. Modal closes.
- **Error toast:** `toast.error(t('contact.error'))` = `"Failed to send. Please try again."`. Modal stays open.
- **Guest behavior:** Modal works without login — prefilled fields are empty strings, no attachment offered.

### D-05: Admin Frequency Trends
- **Card placement:** New card below the activity table in `OverviewTab`. Same card style: `rounded-2xl border bg-white dark:bg-slate-900 p-6 shadow-sm`.
- **Card header:** `<h3 className="font-bold text-slate-800 dark:text-white flex items-center gap-2">` with `BarChart3` lucide icon (`w-4 h-4 text-pride-purple`). Label: `{t('admin.frequencyTrends.title')}`.
- **Time window selector:** Reuse the existing days filter from the parent `AdminDashboard` — pass as prop. No new time filter UI needed.
- **Bar chart (`SimpleBarChart`):**
  - Custom SVG, same viewBox pattern as `SimpleLineChart` (`viewBox="0 0 800 {height}"`).
  - Each bar: color `#7b61ff` (`--color-pride-purple`), `opacity-80`, `rx="4"` (4px rounded corners on bars).
  - Bar labels: category name below x-axis, `text-xs text-slate-500 dark:text-slate-400`, truncated at 14 chars.
  - Bar value: displayed above each bar, `text-xs font-bold text-slate-800 dark:text-white`.
  - Props: `data: {category: string, count: number}[]`, `height?: number` (default 200), `color?: string` (default `#7b61ff`).
- **Top-5 phrases per label:** Expandable disclosure per category bar. Trigger: clicking the bar or a chevron button below the bar label.
  - Collapsed state: shows category name + count badge only.
  - Expanded state: ordered list `<ol>` of up to 5 phrases, `text-xs text-slate-600 dark:text-slate-400`, each line `"[phrase]" × [count]`.
  - Use `framer-motion` `AnimatePresence` + `motion.div` with `height: 0 → auto` — matches existing animation pattern.
- **WebSocket auto-refresh indicator:** Small pulsing dot (`w-2 h-2 rounded-full bg-green-400 animate-pulse`) next to the card title when WebSocket is connected. Disappears when disconnected.
- **Empty state:** Centered text `text-slate-500` — `{t('admin.frequencyTrends.noData')}` = `"No findings recorded in this period."` — same pattern as `OverviewTab` activity empty state.
- **Loading state:** Skeleton pulse (`animate-pulse bg-slate-200 dark:bg-slate-700 rounded`) for the chart area — matching `SkeletonLoader` component pattern from `OverviewTab`.
- **Error state:** `<div className="p-4 bg-red-50 dark:bg-red-900/20 rounded-lg text-red-600 dark:text-red-400 text-sm">Failed to load trends. Please try again.</div>` — same as KPI error pattern.

## Accessibility Contract

- All form inputs have associated `<label>` elements (explicitly via `htmlFor` or wrapping).
- Required fields show `<span className="text-red-500" aria-hidden="true">*</span>` and the input has `aria-required="true"`.
- `Dialog.Title` and `Dialog.Description` are always present in modals (Radix Dialog requirement for `aria-labelledby` / `aria-describedby`).
- Focus ring: `focus-visible:outline-2 focus-visible:outline-pride-purple outline-offset-2` — enforced globally in `globals.css`.
- WebSocket connection-status dot includes `aria-label="Live updates connected"` / `"Live updates disconnected"` on the containing element.
- `HealthWarningBanner` uses `role="alert"` implicitly via `motion.div` (add `role="alert"` explicitly in the D-02 conditional render if not already present on the component).
- Navbar Contact Us button: minimum height 44px (WCAG 2.5.5 AA touch target compliance), `aria-haspopup="dialog"`.

## RTL / Bidirectional Layout

All Phase 8 components render under `html[dir="rtl"]` for Hebrew locale. Constraints:

- Use Tailwind logical properties where available: prefer `ps-` / `pe-` over `pl-` / `pr-` for padding, `ms-` / `me-` for margin.
- `ContactModal`: form layout is vertical (stack) — no horizontal reversal issues. Read-only prefilled row uses `flex items-center gap-2` (direction-aware).
- `SimpleBarChart`: SVG is direction-agnostic; bar labels use `text-anchor="middle"` on `<text>` elements — no RTL issue.
- `FrequencyTrendsCard` header icon + text: use `flex items-center gap-2` — naturally reverses in RTL.
- `HealthWarningBanner`: icon + text row uses `flex items-center gap-2` — reverses correctly.
- Hebrew font (`--font-noto-hebrew`) is already loaded and applied via `html[lang="he"] .hebrew-hero`. New components do not need to set font explicitly — the body inherits the correct font stack.

## Animation Contract

All animations use `framer-motion`. Follow established patterns:

| Animation | Pattern | Where |
|-----------|---------|-------|
| Modal entry | `initial={{ opacity: 0, scale: 0.95 }} animate={{ opacity: 1, scale: 1 }} transition={{ duration: 0.2 }}` | `ContactModal` overlay + content |
| Banner entry | `initial={{ opacity: 0, y: -10 }} animate={{ opacity: 1, y: 0 }}` | `HealthWarningBanner` (already implemented) |
| Card entry | `initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.4 }}` | `FrequencyTrendsCard` on mount |
| Phrase disclosure | `AnimatePresence` + `motion.div` height 0 → auto | Top-5 phrase expandable rows |
| KPI card style | `initial={{ opacity: 0, scale: 0.9, y: 20 }} animate={{ opacity: 1, scale: 1, y: 0 }}` | Already in `OverviewTab` — do NOT replicate in `FrequencyTrendsCard` (single entry animation per card) |
