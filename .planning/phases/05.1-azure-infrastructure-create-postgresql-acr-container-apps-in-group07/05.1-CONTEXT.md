<decisions>
## Implementation Decisions

### Resource Group
- Deploy everything to Group07 (same RG as vLLM VM)
- Single resource group simplifies networking and organization
- No cross-RG VNet peering needed

### VNet Discovery
- Script discovers VNet name from InclusifyModel VM dynamically
- VM name: InclusifyModel (in Group07)
- Use `az vm show` to get network interface, then extract VNet from subnet ID
- Avoids hardcoding VNet name that might be wrong

### Resource Naming
- ACR name: inclusifyacr (current default, keep simple)
- PostgreSQL server: inclusify-db (current default)
- Container Apps Environment: inclusify-env (current default)
- All names are globally unique Azure resources — script should handle name conflicts gracefully

### Script Modifications
- Change default RESOURCE_GROUP from inclusify-rg to Group07
- Change VLLM_RESOURCE_GROUP default to Group07 (same RG now)
- Add VNet discovery logic instead of hardcoded VLLM_VNET_NAME
- Keep idempotent provisioning pattern (existing error handling)

### Claude's Discretion
- Exact VNet discovery commands and parsing
- Error messages and validation logic
- Subnet address prefix selection within VNet address space
- Output formatting and summary

</decisions>
