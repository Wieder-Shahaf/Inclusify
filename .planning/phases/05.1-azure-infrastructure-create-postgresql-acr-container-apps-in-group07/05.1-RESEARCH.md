## Summary

This phase modifies the existing `azure-setup.sh` script to deploy all Azure resources (PostgreSQL, ACR, Container Apps Environment) to Group07 instead of creating a separate resource group. The key technical challenge is dynamically discovering the VNet configuration from the InclusifyModel VM rather than hardcoding VNet names.

The existing script is well-structured with idempotent provisioning patterns. The modifications are straightforward: update default variables, add VNet discovery logic using Azure CLI, and ensure the Container Apps subnet requirements are met.

**Primary recommendation:** Use `az vm show` to get the VM's NIC ID, then `az network nic show` to get the subnet ID, and parse the VNet name from the subnet resource ID using bash string manipulation.

## Architecture Patterns

### VNet Discovery Pattern
**What:** Dynamically discover VNet configuration from existing VM
**When to use:** When VNet names aren't known at script time
**Example:**
```bash
# Source: Microsoft Learn - Azure CLI VM Tutorial
# Step 1: Get NIC ID from VM
NIC_ID=$(az vm show \
  --name "InclusifyModel" \
  --resource-group "Group07" \
  --query 'networkProfile.networkInterfaces[0].id' \
  -o tsv)

# Step 2: Get subnet ID from NIC
SUBNET_ID=$(az network nic show \
  --ids "$NIC_ID" \
  --query 'ipConfigurations[0].subnet.id' \
  -o tsv)

# Step 3: Parse VNet name from subnet ID
# Format: /subscriptions/{sub}/resourceGroups/{rg}/providers/Microsoft.Network/virtualNetworks/{vnetName}/subnets/{subnetName}
VNET_NAME=$(echo "$SUBNET_ID" | cut -d'/' -f9)
```

### Container Apps Subnet Creation
**What:** Create dedicated subnet with proper delegation for Container Apps
**When to use:** When integrating Container Apps with existing VNet
**Requirements:**
- Minimum /27 CIDR for workload profiles environment
- Must delegate to `Microsoft.App/environments`
- Subnet cannot be used by other services

**Example:**
```bash
# Source: Microsoft Learn - Container Apps VNet integration
az network vnet subnet create \
  --resource-group "Group07" \
  --vnet-name "$VNET_NAME" \
  --name "containerapps-subnet" \
  --address-prefixes "10.0.2.0/27" \
  --delegations Microsoft.App/environments \
  --output none 2>/dev/null || echo "Subnet already exists, skipping"
```

### Existing Script Patterns to Preserve
The current `azure-setup.sh` uses these patterns:
- **Idempotent provisioning:** `|| echo "already exists, skipping"`
- **Environment variable overrides:** `${VAR:-default}` pattern
- **9-step numbered output:** Visual progress indicators
- **Required secret validation:** Exit early if PG_PASSWORD missing
- **Connection summary:** Print all connection info at end

### Anti-Patterns to Avoid
- **Hardcoding VNet names:** VM may be in differently-named VNet than expected
- **Assuming subnet availability:** The VNet address space may not have room for the requested prefix
- **Silent failures:** Always capture and report az command failures

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| VNet name extraction | Complex regex parsing | `cut -d'/' -f9` on subnet ID | Subnet ID format is stable, simple parsing works |
| Subnet address selection | IP range calculator | Hardcode sensible default (10.0.2.0/27) | Existing VNet uses 10.0.x.0/24, so 10.0.2.0/27 is safe |
| Azure resource validation | Custom API calls | `az resource show --ids` with `2>/dev/null` | Built-in CLI handles all edge cases |

**Key insight:** The subnet ID from Azure CLI already contains all needed information in a predictable format. No need for complex API calls or parsing.

## Common Pitfalls

### Pitfall 1: VNet Discovery Fails If VM Has No NIC
**What goes wrong:** `az vm show` returns empty networkInterfaces array
**Why it happens:** VM might not have a NIC attached (unlikely for InclusifyModel)
**How to avoid:** Check for empty NIC_ID before proceeding
**Warning signs:** Empty string from NIC_ID query

### Pitfall 2: Subnet Address Conflict
**What goes wrong:** Subnet creation fails with "address range overlaps"
**Why it happens:** Requested subnet prefix overlaps with existing subnets
**How to avoid:** Check existing subnets first, or handle error gracefully
**Warning signs:** Azure error mentioning "address space" or "overlaps"

### Pitfall 3: Container Apps Environment Creation in Wrong RG
**What goes wrong:** Environment created in Group07 but subnet is in different RG
**Why it happens:** VNet/subnet must be in same region, cross-RG networking has restrictions
**How to avoid:** Since everything is in Group07, this is already handled correctly
**Warning signs:** N/A - current design avoids this

### Pitfall 4: Global Resource Name Conflicts
**What goes wrong:** ACR or PostgreSQL server name already taken globally
**Why it happens:** These names must be globally unique across all Azure
**How to avoid:** Handle "already exists" errors gracefully (idempotent pattern)
**Warning signs:** Error mentioning "name already in use" or "conflict"

### Pitfall 5: Missing Subnet Delegation
**What goes wrong:** Container Apps environment creation fails
**Why it happens:** Subnet must be delegated to `Microsoft.App/environments` for workload profiles
**How to avoid:** Always include `--delegations Microsoft.App/environments` when creating subnet
**Warning signs:** Error mentioning "delegation" or "Microsoft.App/environments"

## Code Examples

### Complete VNet Discovery Function
```bash
# Source: Verified pattern from Microsoft Learn docs
discover_vnet_from_vm() {
  local vm_name=$1
  local resource_group=$2

  echo "Discovering VNet from VM '$vm_name' in '$resource_group'..."

  # Get NIC ID
  local nic_id
  nic_id=$(az vm show \
    --name "$vm_name" \
    --resource-group "$resource_group" \
    --query 'networkProfile.networkInterfaces[0].id' \
    -o tsv 2>/dev/null)

  if [ -z "$nic_id" ]; then
    echo "Error: Could not find network interface for VM '$vm_name'"
    return 1
  fi

  # Get subnet ID
  local subnet_id
  subnet_id=$(az network nic show \
    --ids "$nic_id" \
    --query 'ipConfigurations[0].subnet.id' \
    -o tsv 2>/dev/null)

  if [ -z "$subnet_id" ]; then
    echo "Error: Could not find subnet for NIC"
    return 1
  fi

  # Parse VNet name (position 9 in /-separated path)
  local vnet_name
  vnet_name=$(echo "$subnet_id" | cut -d'/' -f9)

  echo "  Discovered VNet: $vnet_name"
  echo "$vnet_name"
}
```
