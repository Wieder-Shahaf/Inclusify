#### 1. Full OAuth Flow Integration Test

**Test:**
1. Set up Google OAuth credentials in Google Cloud Console:
   - Create OAuth 2.0 Client ID at https://console.cloud.google.com/apis/credentials
   - Add authorized redirect URI: http://localhost:8000/auth/google/callback
   - Copy Client ID and Client Secret
2. Set environment variables:
   ```bash
   export GOOGLE_CLIENT_ID="your-client-id.apps.googleusercontent.com"
   export GOOGLE_CLIENT_SECRET="your-client-secret"
   export FRONTEND_URL="http://localhost:3000"
   ```
3. Start backend: `cd backend && uvicorn app.main:app --reload --port 8000`
4. Visit http://localhost:8000/auth/google/authorize
5. Complete Google OAuth consent flow
6. Verify redirect to http://localhost:3000/en/oauth/callback?access_token=...

**Expected:**
- Redirect to Google consent screen with scopes: openid, email, profile
- After consent, redirect back to callback with authorization code
- Backend exchanges code for token, creates/links user, redirects to frontend with JWT
- JWT can be decoded and contains user_id, role, and standard FastAPI Users claims

**Why human:**
External service integration with real Google APIs. Cannot mock Google's consent UI and token exchange in automated tests. Requires manual setup of Google Cloud project and credentials.
