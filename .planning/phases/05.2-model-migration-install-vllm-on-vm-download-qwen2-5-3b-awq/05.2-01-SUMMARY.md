## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Cleared disk space for vLLM installation**
- **Found during:** Task 1
- **Issue:** VM disk was 95% full (only 3.3GB free), vLLM installation failed with "No space left on device"
- **Fix:** Cleared pip cache (16GB recovered), freed up 25GB total
- **Impact:** None - cleanup only

**2. [Rule 3 - Blocking] Downgraded transformers for vLLM compatibility**
- **Found during:** Task 2
- **Issue:** vLLM 0.6.6 incompatible with transformers 5.x (AttributeError: `all_special_tokens_extended` removed)
- **Fix:** Downgraded transformers from 5.3.0 to 4.57.6
- **Impact:** None - version pinning for compatibility

## Decisions Made

1. **Dedicated vllm-venv:** Created separate virtual environment `/home/azureuser/vllm-venv/` to avoid conflicts with existing environments (`~/dev/venv/`, `~/inclusify/.venv/`)

2. **transformers 4.x:** Pinned to `transformers>=4.45.2,<5.0` because vLLM 0.6.6 uses deprecated `all_special_tokens_extended` attribute removed in transformers 5.x

3. **Model path as model name:** vLLM serves the model using the full directory path as the model identifier: `/home/azureuser/models/Qwen2.5-3B-Instruct-GPTQ-Int4`
