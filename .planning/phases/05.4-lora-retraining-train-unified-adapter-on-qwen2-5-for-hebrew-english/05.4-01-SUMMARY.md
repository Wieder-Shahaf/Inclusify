## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed duplicate sentences in dataset**
- **Found during:** Task 2 (test_stratified_split failing)
- **Issue:** Dataset contained 12 duplicate sentences causing data leakage during train/val split
- **Fix:** Added deduplication in `load_dataset()` using `df.drop_duplicates(subset=["Sentence"], keep="first")`
- **Files modified:** `ml/training/prepare_data.py`
- **Impact:** Reduced dataset from 1000 to 988 unique samples, eliminated data leakage
- **Commit:** 22cdeac (included in Task 2 commit)

**Why Rule 1:** Data leakage is a bug (incorrect behavior) that compromises training validity. Deduplication is the standard fix for this correctness issue.

## Key Decisions Made

1. **Deduplication strategy:** Remove duplicates during load (not after split) to prevent any possibility of leakage
2. **Keep-first policy:** Preserve first occurrence of duplicates to maintain original dataset ordering
3. **Test tolerance:** 5% tolerance for class proportion matching (actual deviation ≤0.3%)
4. **VM environment reuse:** Install in existing vllm-venv rather than creating new environment
5. **System prompt simplicity:** Focused prompt on severity classification task (not category prediction)
