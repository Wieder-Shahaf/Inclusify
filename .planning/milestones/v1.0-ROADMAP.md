## Overview

Inclusify v1.0 transforms a working POC with demo data into a production-deployed LGBTQ+ inclusive language analyzer. Starting from a blank infrastructure, the milestone activated the existing DB layer, containerized all services, deployed the fine-tuned Qwen2.5-3B model on Azure GPU via vLLM, wired the frontend to the real API, added authentication, RBAC, Google OAuth, an admin dashboard, accessibility compliance, and shipped a GitHub Actions CI/CD pipeline — hitting the April 15, 2026 E2E demo deadline.

## Milestone Summary

### Decimal Phases (Urgent Insertions)

- Phase 5.1: Azure Infrastructure — Group07 resource group, dynamic VNet discovery
- Phase 5.2: Model Migration — GPTQ over AWQ (T4 compute 7.5 incompatibility discovered)
- Phase 5.3: Auth Frontend — Full auth UI with Google OAuth, guest mode
- Phase 5.4: LoRA Retraining — Qwen2.5 adapter validated with 90% F1
- Phase 5.4.1: Dataset Synthesis — 10K English + Hebrew dataset pipeline
- Phase 5.5: Backend OAuth — Google OAuth server-side flow with auto-link

### Key Decisions

- Docker: Python 3.12-slim and Node 22-slim (not Alpine) for native module compatibility
- Azure: CLI scripts over Terraform (course simplicity), Group07 resource group
- Auth: FastAPI Users 13.x + pwdlib (passlib deprecated), Redis-backed refresh
- LLM: GPTQ-Int4 over AWQ (T4 GPU compute 7.5 incompatibility), vLLM 0.6.6
- Hybrid detection: LLM preferred over rules for overlapping spans (50% threshold)
- Frontend demo mode: NEXT_PUBLIC_USE_DEMO_MODE toggle for gradual rollout
- OAuth: Backend-only code exchange with JWT redirect to frontend
- Admin analytics: Raw asyncpg SQL (not ORM) for aggregate queries
- Accessibility: LiveAnnouncer with 50ms clear-then-set, RTL arrow key flip
- Private mode: Default OFF per user decision

### Issues Resolved

- AWQ incompatible with T4 GPU → switched to GPTQ-Int4
- vLLM 0.17+ requires CUDA 12.9+ → pinned to 0.6.6
- passlib deprecated → migrated to pwdlib
- Docling memory exhaustion → subprocess isolation
- Phase 5.5 OAuthAccount FK → overrode user_id to point to 'users' table
- vLLM response parsing crash → safe bounds checking in llm_client.py

### Technical Debt

- Phase 03-01 (vLLM VM deployment) executed manually; no automation script for full VM bootstrap
- Phase 5.4.1 dataset synthesis blocked on API key availability during initial execution
- REQUIREMENTS.md traceability table not updated in real-time with ROADMAP.md checkboxes
