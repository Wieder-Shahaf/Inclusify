## Milestone Summary

**Key Decisions:**

- ProfileSetupModal completion check requires all 3 fields (full_name AND institution AND profession)
- dismiss() used in onSubmit success to prevent race condition on refreshProfile() re-render
- he.json profile.setup.skip corrected to "דלג בינתיים" per UI-SPEC copywriting contract
- Option B thin harness used for TDD test — PaperUpload two-step flow made full-page integration test impractical
- HealthWarningBanner variant="error" for rules_only results banner per UI-SPEC
- Footer watermark at pageHeight-8mm centered, gray 8pt; locale switch he → Hebrew string
- returnBase64 uses doc.output('datauristring'); no doc.save() call when true
- Recipients always queried from DB WHERE role='site_admin'; POST body sender_email never used for routing
- AnalysisData defined locally in ContactModal.tsx (not re-exported from exportReport.ts)
- models.py str | None → Optional[str] for Python 3.9 SQLAlchemy compatibility
- WebSocket JWT auth via query param — Depends() injection does not work in FastAPI WS handlers
- AdminWSManager as module-level singleton in admin/router.py; imported by analysis/router.py
- Broadcast wrapped in try/except so WS failures never fail the analysis request
- WS close code 4001 for missing/invalid token; 4003 for non-admin role
- Custom SVG SimpleBarChart — no D3 dependency, follows existing SimpleLineChart pattern

**Issues Resolved:**

- SMTP env vars missing from docker-compose.yml — fixed by adding SMTP_USER/PASSWORD/HOST/PORT
- Security leak: SMTP exception details were leaking to clients — fixed with generic error response
- Status badge using 'completed' instead of 'succeeded' — corrected
- JWT-in-URL security limitation documented for admin WebSocket
- private_mode fallback corrected to False

**Issues Deferred:**

- D-05 WebSocket auto-refresh UAT test pending (human testing required for cross-tab scenario)
- Phase 01, 03, 05.2, 08 VERIFICATION.md items deferred as human_needed
- Quick task 260407-kps missing SUMMARY.md
